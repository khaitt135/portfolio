Phase 1 — The Full Automation
n8n + OpenAI API · HubSpot · Google Docs

The original build: a fully autonomous n8n workflow triggered from HubSpot, calling the OpenAI API to generate brand onboarding documents, and writing the output back to Google Docs and HubSpot. No human needed at execution time — only at review. Shut down when API token costs became unsustainable for a seed-stage startup with low usage volume.

Why it stopped
Cost architecture didn't fit the usage pattern
OpenAI API charges per token — every run costs real money regardless of output value
Seed-stage startup = low run volume — cost-per-doc was too high to justify vs. manual
n8n itself is free (self-hosted) — the orchestration layer wasn't the problem
Decision: keep the workflow logic, replace the API layer with a no-cost alternative
Phase 2 — The Rebuild
Custom GPT + GPT Actions · no per-token cost

Rebuilt as a Custom GPT inside ChatGPT (Plus/Business plan) — trained on 250+ structured brand data inputs, with a detailed system prompt encoding the onboarding template logic. GPT Actions wired to HubSpot and Google Docs for output delivery. The tradeoff: a human must initiate each session inside ChatGPT. No background scheduling. No autonomous trigger. The AI does the heavy generation work; the operator navigates the conversation.

Learnings
💰
Architecture is a cost decision first
The technically superior solution (autonomous n8n + API) was wrong for a startup with 5 onboardings a month. The simpler solution (Custom GPT) was right. Picking the right architecture means understanding usage volume, not just capability. Over-engineering is a real cost.
💡
Custom GPT ≠ automation — it's an AI-assisted tool
A Custom GPT requires a human to open, initiate, and guide each session. It is not a background process. The time savings come from AI doing the generation work — not from removing human involvement. Understanding this distinction matters when scoping what an AI tool can actually do.
🔨
Prompt engineering is the persistent layer
Both phases required the same core skill: writing prompts that reliably produce structured, on-brand onboarding documents. Whether that prompt lives in an n8n node or a Custom GPT system prompt is an implementation detail. The intellectual work — structuring the output, training on 250+ data points, tuning for consistency — transferred completely across both architectures.
