Reflection & Learnings
üîß
End-to-end ML ownership
Building solo ‚Äî from raw CSV to deployed app ‚Äî forced mastery of the full pipeline. Data cleaning, feature engineering, model selection, evaluation, serialisation, and deployment are all distinct skill sets. Doing all of them in one project was the real learning.
üß©
Feature mismatches are the #1 deployment bug
The biggest technical challenge was ensuring features fed into the live app exactly matched the training schema. Column order, encoding format, and scaling parameters all had to be preserved through Joblib ‚Äî a non-obvious but critical deployment detail.
üìä
Model selection is a business decision
HistGradientBoosting won on R¬≤. But for a real insurance product, interpretability might outweigh accuracy ‚Äî a Linear Regression at 0.77 that a regulator can audit may be more valuable than a 0.863 black box. The right model depends on the use case.
üöÄ
Deployment is where value is created
A notebook result nobody can use is not a product. Building the Streamlit app turned a model into something anyone can interact with. Deployment is the step most ML courses skip. It shouldn't be.
